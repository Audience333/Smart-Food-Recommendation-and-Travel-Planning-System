# 智慧美食推荐与漫游规划系统

高德地图 Web 可视化配合 C++ 数据管道

## 项目简介

采集菏泽市 288 条真实美食门店和 110 个景点数据作为基础数据源，通过 C++ 命令行管道对接高德地图 API 完成数据获取和加工（含坐标采集、评分提取、地址补全、营业时间解析、门店照片拉取），前端基于原生 JavaScript 和高德 JSAPI 2.0 构建完整的交互式地图应用，提供搜索筛选、驾车和步行路线规划、收藏管理、用户画像分析、一日游智能推荐、多维度排行榜、操作历史撤销重做等功能。前后端均自实现决策树、顺序栈、双向链表、邻接表无向图、Dijkstra 最短路径（含手写小顶堆）、BFS 广度优先搜索共六种数据结构。

## 技术栈

数据管道使用 C++17 标准，通过命令行 curl 发起 HTTP 请求调用高德 Web API，采用定向字符串匹配提取 JSON 字段，零外部库依赖，单次 g++ 编译即可使用。Web 前端采用原生 JavaScript 配合高德地图 JSAPI 2.0，所有交互逻辑集中在一个 JavaScript 文件中。数据以管道分隔的 TXT 格式存储于本地，再转换为 JSON 供前端加载，用户状态通过浏览器 localStorage 持久化。

## 数据管道

管道由六个子命令组成，通过 pipeline.exe 加子命令名调用。全部执行时会按 expand、fill、photos、roads、json 的顺序依次完成数据采集到前端格式转换的全流程。

expand 命令调用高德地点搜索接口（v3 place text），按 9 个区县加 16 个美食关键词和 7 个景点关键词分别搜索，每页取 20 条翻 2 页，对返回的 POI 结果做名称去重和坐标距离去重（200 米以内且相似度超过七成视为重复），然后为每条美食生成 12 个维度的自动标签（口味口感、食材主料、烹饪方式、菜系流派、用餐时段、用餐场景、服务设施、荣誉标签、价格档次、门店状态、适合人群、特色标记），按评分排序取各区前 25 条美食和前 10 个景点，追加写入 TXT 源文件。

fill 命令调用高德逆地理编码接口（v3 geocode regeo），找出所有地址字段为空的 POI，按坐标去重后逐一请求获取真实地址并回填到 TXT 文件，坐标完全相同的 POI 共享一次请求结果。photos 命令先通过地点搜索找到 POI 的高德 ID，再调用地点详情接口（v3 place detail）获取每个 POI 最多三张门店照片的 URL。roads 命令使用手写邻接表图构建 398 个顶点和 2890 条边的城市路网，在景点间使用高德驾车路径规划 API 计算真实道路距离，同一区县内的美食间使用 Haversine 球面距离公式计算。json 命令将管道分隔的 TXT 源数据解析为前端可直接加载的 JSON 数组格式。

## 手写数据结构的实际使用

项目中自实现的六种数据结构全部在前后端代码中有实际调用，以下是各结构对应的代码位置。

决策树用于多条件美食筛选。C++ 端在 pipeline.cpp 的 cmd_expand 函数中第 1108 行，实例化 DecisionTree 并链式添加评分不低于 3.5 和名称长度不少于 2 两条条件，对候选 POI 做二次过滤。JavaScript 端在 map.js 的 showFoodMarkers 函数中第 1790 行，调用 build 方法根据当前活跃的分类集合和仅显示营业中的开关状态构建筛选条件链，然后对每道美食调用 root.evaluate 判断是否通过。

顺序栈用于操作历史的撤销重做。C++ 端在 pipeline.cpp 的 cmd_expand 函数中第 1064 行，实例化 SeqStack 暂存处理中的 POI 名称。JavaScript 端在 map.js 的 HistoryManager 中全面接管了操作记录，push 方法在第 239 行记录新操作，undo 在第 248 行回退，redo 在第 257 行前进，canUndo 和 canRedo 在第 264 和 269 行判断可用状态，getAll 和 getCurrent 在第 391 和 392 行为历史面板提供数据，clear 在第 406 行清空。

双向链表用于收藏夹管理。C++ 端在 pipeline.cpp 的 cmd_expand 函数中第 1031 行，实例化 DoublyLinkedList 替代原有的 vector 容器存储新增的美食行和景点行。JavaScript 端在 map.js 的 FavoriteManager 中全面接管了收藏夹，pushBack 在第 450 行尾插新收藏，remove 在第 464 行按条件删除节点，contains 在第 475 行检查是否存在，forwardTraverse 在第 479 行和第 645 行分别用于获取所有收藏列表和统计各美食的收藏数供排行榜使用，clear 在第 498 行清空后从 localStorage 恢复。

邻接表图用于城市路网搭建。C++ 端在 pipeline.cpp 的 cmd_roads 函数中第 1764 行，将所有美食和景点的坐标作为顶点加入图中，将从 road.txt 解析的边逐一添加为无向边。JavaScript 端在 map.js 的 loadData 函数中，从 road.txt 读取边数据构建同样结构的路网图并实例化 Dijkstra 和 BFS。

Dijkstra 最短路径算法用于路线规划的降级方案。C++ 端在 pipeline.cpp 的 cmd_roads 函数中第 1993 行，当高德驾车 API 不可用时使用手写小顶堆优化的 Dijkstra 算法在图中寻找景点间的最短路径，并还原完整路径节点序列。JavaScript 端在 map.js 的 renderRouteSegmentFallback 函数中，在 Haversine 直线距离之前先尝试用 Dijkstra 在图路网中寻找最短路径。

BFS 广度优先搜索用于周边美食发现。C++ 端在 pipeline.cpp 的 cmd_roads 函数中第 2058 行，对 9 个核心景点各执行 BFS 搜索其 5 公里范围内的所有美食 POI 并输出聚类结果。JavaScript 端在 map.js 的 renderRouteSummary 函数中，在路线规划完成后搜索终点周边 5 公里内的美食并展示在路线摘要面板中。

## Web 前端功能

地图上以不同颜色的圆形标记展示美食，蓝色方形标记展示景点，标记上显示菜名首字或景字。顶部标题栏集成撤销重做按钮和操作历史下拉面板，以及收藏夹和排行榜的悬浮面板，光标点击即可展开。

图层控制使用按钮式开关分别控制美食和景点的显示隐藏，另有仅显示营业中按钮根据当前时间与各门店的营业时段做实时比对后筛选。美食分类面板列出汤类、面食、小吃、正餐、烧烤、甜品、饮品、凉菜八个分类标签，支持多选筛选，选中后地图自动缩放至匹配标记的视野范围。

搜索功能在侧边栏输入关键词即可对美食和景点的名称、地址、标签做实时模糊匹配，搜索结果以列表展示，点击任一结果地图跳转并弹出详情窗口，未匹配的标记降低透明度以突出搜索结果。

路线规划面板支持在驾车和步行两种通行方式间切换，起点和终点的选择器上方各有一个搜索输入框可实时过滤数百个 POI 选项，途经点最多添加五个并支持删除和排序，排序策略提供时间优先、距离优先和过路费少三种。规划路线后通过高德路径规划接口获取真实道路距离和行驶时间，结果在面板中分总览和分段详情展示，同时在地图上以渐变色折线绘制路径，每段中央标注距离和时间。

用户画像面板根据收藏夹中的美食自动分析口味标签云、价格区间分布条形图和品类偏好占比图，支持编辑模式下手动加减各标签的权重值、从系统中已有标签列表选择新增、或一键恢复系统自动总结的原始偏好。

一日游推荐面板基于用户画像中的偏好品类和价格区间，智能匹配高分美食并自动寻找最近的景点作为游览起点，生成三个候选方案并展示各方案的预估距离和评分，换一批按钮可刷新获取新的组合，采纳按钮一键将选中方案填入路线规划面板。

排行榜面板提供人气榜按评分降序、性价比榜按评分与价格的自然对数比降序、收藏榜按被收藏次数降序三种维度，前三位显示奖牌图标，支持升序降序切换，每项均可点击跳转地图定位。

收藏夹数据存储在浏览器 localStorage 中，收藏的美食在地图上以金色边框标记区分。详情浮窗展示评分星级、人均价格、分类、地址、营业时间，以及高德获取的门店真实照片和十二维度标签。

## 快速开始

获取项目后直接双击运行系统.bat（Windows）或在终端执行 bash 运行系统.sh（Linux/Mac），脚本会自动检测本机 Python 环境、启动静态文件服务器并打开浏览器访问 localhost 的 8080 端口。

如需重新生成数据，先确保系统已安装 g++ 和 curl，进入 HezeFoodSystem 目录，执行 g++ std c++17 O2 tools_cpp pipeline.cpp o tools_cpp pipeline.exe 编译管道程序，然后可按需运行 pipeline expand（扩充数据）、pipeline fill（补全地址）、pipeline photos（获取照片）、pipeline roads（计算道路）、pipeline json（生成前端 JSON），或直接 pipeline all 一键执行全部。

## 数据概况

当前系统包含美食门店 288 条（其中 63 条为原始数据、225 条通过高德 API 扩充），景点 110 处（20 处原始、90 处扩充），道路连接 2890 条。各 POI 均包含真实坐标、评分、人均价格、地址、营业时间和至少三张门店照片，照片覆盖率达到百分之九十六点七，地址覆盖率百分之百。每道美食携带十二个维度的自动标签，平均每店 16 条标签。数据覆盖菏泽市全部九个区县，分别是牡丹区、单县、曹县、郓城县、巨野县、东明县、定陶区、成武县、鄄城县。
