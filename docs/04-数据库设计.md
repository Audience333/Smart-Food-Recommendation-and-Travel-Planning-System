# 菏泽美食推荐与城市漫游规划系统 - 数据库设计

## 一、数据库概述

### 1.1 数据库选型
- **主数据库**：MySQL 8.0
- **字符集**：utf8mb4（支持中文和 emoji）
- **排序规则**：utf8mb4_unicode_ci

### 1.2 设计原则
- 遵循第三范式（3NF），减少数据冗余
- 合理使用 JSON 类型存储灵活数据
- 预留扩展字段，支持未来功能迭代

## 二、表结构设计

### 2.1 美食表（food）

```sql
CREATE TABLE food (
    id              INT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    name            VARCHAR(100) NOT NULL COMMENT '美食名称',
    category        VARCHAR(50) NOT NULL COMMENT '美食分类',
    description     TEXT COMMENT '美食描述',
    image_url       VARCHAR(500) COMMENT '图片URL',
    longitude       DECIMAL(10, 7) NOT NULL COMMENT '经度',
    latitude        DECIMAL(10, 7) NOT NULL COMMENT '纬度',
    address         VARCHAR(200) NOT NULL COMMENT '详细地址',

    -- 特色信息
    cuisine         VARCHAR(50) COMMENT '菜系',
    signature_dish  VARCHAR(100) COMMENT '招牌菜',
    price_range     VARCHAR(20) COMMENT '价格区间（如：20-50元）',
    price_level     TINYINT COMMENT '价格等级（1-5）',
    score           DECIMAL(2, 1) DEFAULT 0 COMMENT '评分（1.0-5.0）',
    review_count    INT DEFAULT 0 COMMENT '评价数量',

    -- 推荐特征（用于推荐算法）
    spicy_level     TINYINT DEFAULT 0 COMMENT '辣度（0-5）',
    sweet_level     TINYINT DEFAULT 0 COMMENT '甜度（0-5）',
    salty_level     TINYINT DEFAULT 0 COMMENT '咸度（0-5）',
    sour_level      TINYINT DEFAULT 0 COMMENT '酸度（0-5）',
    texture         VARCHAR(50) COMMENT '口感（酥脆/软糯/爽滑等）',
    environment     VARCHAR(50) COMMENT '环境特征',
    suitable_scene  VARCHAR(100) COMMENT '适合场景（早餐/午餐/夜宵等）',
    target_group    VARCHAR(100) COMMENT '目标用户群体',

    -- 标签（JSON 数组格式）
    tags            JSON COMMENT '标签集合',

    -- 状态字段
    status          TINYINT DEFAULT 1 COMMENT '状态（0-下架 1-正常）',
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',

    -- 索引
    INDEX idx_category (category),
    INDEX idx_score (score),
    INDEX idx_location (latitude, longitude),
    INDEX idx_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='美食表';
```

### 2.2 景点表（spot）

```sql
CREATE TABLE spot (
    id              INT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    name            VARCHAR(100) NOT NULL COMMENT '景点名称',
    description     TEXT COMMENT '景点描述',
    image_url       VARCHAR(500) COMMENT '图片URL',
    longitude       DECIMAL(10, 7) NOT NULL COMMENT '经度',
    latitude        DECIMAL(10, 7) NOT NULL COMMENT '纬度',
    address         VARCHAR(200) NOT NULL COMMENT '详细地址',

    -- 景点信息
    type            VARCHAR(50) NOT NULL COMMENT '景点类型',
    ticket_info     VARCHAR(100) COMMENT '门票信息',
    opening_time    VARCHAR(100) COMMENT '开放时间',
    recommend_duration VARCHAR(50) COMMENT '建议游玩时长',
    best_season     VARCHAR(50) COMMENT '最佳季节',

    -- 评分
    score           DECIMAL(2, 1) DEFAULT 0 COMMENT '评分（1.0-5.0）',
    review_count    INT DEFAULT 0 COMMENT '评价数量',

    -- 标签
    tags            JSON COMMENT '标签集合',

    -- 状态字段
    status          TINYINT DEFAULT 1 COMMENT '状态（0-关闭 1-正常）',
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',

    -- 索引
    INDEX idx_type (type),
    INDEX idx_score (score),
    INDEX idx_location (latitude, longitude),
    INDEX idx_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='景点表';
```

### 2.3 用户表（user）

```sql
CREATE TABLE user (
    id              INT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    username        VARCHAR(50) NOT NULL UNIQUE COMMENT '用户名',
    password_hash   VARCHAR(255) NOT NULL COMMENT '密码哈希',
    nickname        VARCHAR(50) COMMENT '昵称',
    avatar          VARCHAR(500) COMMENT '头像URL',
    email           VARCHAR(100) COMMENT '邮箱',
    phone           VARCHAR(20) COMMENT '手机号',

    -- 用户画像
    taste_preference JSON COMMENT '口味偏好（辣/甜/咸/酸的偏好程度）',
    consumption_level VARCHAR(20) COMMENT '消费水平（low/mid/high）',
    travel_preference JSON COMMENT '游玩偏好',
    preferred_cuisines JSON COMMENT '喜欢的菜系',

    -- 统计信息
    browse_count    INT DEFAULT 0 COMMENT '浏览次数',
    favorite_count  INT DEFAULT 0 COMMENT '收藏数量',
    comment_count   INT DEFAULT 0 COMMENT '评论数量',

    -- 状态字段
    status          TINYINT DEFAULT 1 COMMENT '状态（0-禁用 1-正常）',
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    last_login_at   DATETIME COMMENT '最后登录时间',

    -- 索引
    INDEX idx_username (username),
    INDEX idx_email (email),
    INDEX idx_phone (phone)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

### 2.4 评论表（comment）

```sql
CREATE TABLE comment (
    id              INT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    user_id         INT NOT NULL COMMENT '用户ID',
    target_id       INT NOT NULL COMMENT '目标ID（美食或景点）',
    target_type     VARCHAR(20) NOT NULL COMMENT '目标类型（food/spot）',

    -- 评论内容
    content         TEXT NOT NULL COMMENT '评论内容',
    score           TINYINT COMMENT '评分（1-5）',
    images          JSON COMMENT '评论图片URL列表',

    -- 评论来源
    source          VARCHAR(20) DEFAULT 'user' COMMENT '来源（user/system/crawl）',

    -- 状态字段
    status          TINYINT DEFAULT 1 COMMENT '状态（0-删除 1-正常）',
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',

    -- 索引
    INDEX idx_user (user_id),
    INDEX idx_target (target_id, target_type),
    INDEX idx_created (created_at),

    -- 外键
    FOREIGN KEY (user_id) REFERENCES user(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='评论表';
```

### 2.5 收藏表（favorite）

```sql
CREATE TABLE favorite (
    id              INT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    user_id         INT NOT NULL COMMENT '用户ID',
    target_id       INT NOT NULL COMMENT '目标ID（美食或景点）',
    target_type     VARCHAR(20) NOT NULL COMMENT '目标类型（food/spot）',

    -- 收藏备注
    remark          VARCHAR(200) COMMENT '用户备注',

    -- 状态字段
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '收藏时间',

    -- 索引
    INDEX idx_user (user_id),
    INDEX idx_target (target_id, target_type),
    UNIQUE INDEX uk_user_target (user_id, target_id, target_type),

    -- 外键
    FOREIGN KEY (user_id) REFERENCES user(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='收藏表';
```

### 2.6 路线表（route）

```sql
CREATE TABLE route (
    id              INT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    user_id         INT COMMENT '创建用户ID（NULL表示系统生成）',
    name            VARCHAR(100) NOT NULL COMMENT '路线名称',
    description     TEXT COMMENT '路线描述',

    -- 路线信息
    route_type      VARCHAR(20) NOT NULL COMMENT '路线类型（food/spot/mixed）',
    duration        VARCHAR(50) COMMENT '预计时长',
    distance        DECIMAL(10, 2) COMMENT '总距离（公里）',

    -- 路线节点（JSON数组，按顺序存储）
    waypoints       JSON NOT NULL COMMENT '途经点列表',
    -- 格式：[{"id": 1, "type": "food", "name": "xxx", "lng": 115.5, "lat": 35.2}, ...]

    -- 评分
    score           DECIMAL(2, 1) DEFAULT 0 COMMENT '评分',
    use_count       INT DEFAULT 0 COMMENT '使用次数',

    -- 标签
    tags            JSON COMMENT '标签（一日游/半日游/美食专线等）',

    -- 状态字段
    status          TINYINT DEFAULT 1 COMMENT '状态（0-删除 1-正常）',
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',

    -- 索引
    INDEX idx_user (user_id),
    INDEX idx_type (route_type),
    INDEX idx_score (score),

    -- 外键
    FOREIGN KEY (user_id) REFERENCES user(id) ON DELETE SET NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='路线表';
```

### 2.7 用户浏览记录表（browse_history）

```sql
CREATE TABLE browse_history (
    id              INT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    user_id         INT NOT NULL COMMENT '用户ID',
    target_id       INT NOT NULL COMMENT '目标ID',
    target_type     VARCHAR(20) NOT NULL COMMENT '目标类型（food/spot）',
    browse_duration INT DEFAULT 0 COMMENT '浏览时长（秒）',
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '浏览时间',

    -- 索引
    INDEX idx_user (user_id),
    INDEX idx_created (created_at),

    -- 外键
    FOREIGN KEY (user_id) REFERENCES user(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='浏览记录表';
```

### 2.8 刷新令牌表（refresh_token）

```sql
CREATE TABLE refresh_token (
    id              INT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    user_id         INT NOT NULL COMMENT '用户ID',
    token           VARCHAR(500) NOT NULL COMMENT 'Refresh Token',
    device_info     VARCHAR(200) COMMENT '设备信息（User-Agent 摘要）',
    ip_address      VARCHAR(45) COMMENT '登录IP',
    expires_at      DATETIME NOT NULL COMMENT '过期时间',
    revoked         TINYINT DEFAULT 0 COMMENT '是否已吊销（0-有效 1-已吊销）',
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',

    -- 索引
    UNIQUE INDEX uk_token (token),
    INDEX idx_user (user_id),
    INDEX idx_expires (expires_at),

    -- 外键
    FOREIGN KEY (user_id) REFERENCES user(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='刷新令牌表';
```

**用途说明**：
- 存储用户的 Refresh Token，支持 Token 刷新机制
- 支持多设备登录（每个设备一个 Refresh Token）
- 支持主动吊销（修改密码、退出登录时使 Token 失效）
- 定期清理过期记录

### 表关系汇总

|------|------|
| user → comment | 一对多，一个用户可发表多条评论 |
| user → favorite | 一对多，一个用户可收藏多个目标 |
| user → refresh_token | 一对多，一个用户可有多个设备的 Token |
| user → browse_history | 一对多，一个用户有多条浏览记录 |
| user → route | 一对多，一个用户可创建多条路线 |
| food/spot → comment | 一对多，一个美食/景点可有多条评论 |

## 五、数据库优化建议

### 5.1 索引优化
- 经纬度字段使用联合索引，支持范围查询
- 高频查询字段（category, type, score）建立单列索引
- JSON 字段如需频繁查询，考虑提取为独立字段

### 5.2 查询优化
- 使用分页查询，避免一次性加载大量数据
- 热点数据使用 Redis 缓存
- 复杂查询使用数据库视图

### 5.3 数据维护
- 定期清理软删除数据
- 定期分析慢查询日志
- 监控数据库性能指标
