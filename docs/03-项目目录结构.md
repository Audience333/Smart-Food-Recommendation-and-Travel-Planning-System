## 二、目录功能说明

### 2.1 frontend/ - 前端项目
| 目录 | 作用 |
|------|------|
| `src/api/` | 封装所有后端 API 请求，统一管理接口调用 |
| `src/components/` | 可复用的 Vue 组件，按功能分组 |
| `src/composables/` | Vue 3 组合式函数，抽取可复用的业务逻辑 |
| `src/views/` | 页面级组件，按功能模块分目录 |
| `src/router/` | Vue Router 路由配置和路由守卫 |
| `src/stores/` | Pinia 状态管理，按业务模块拆分 |
| `src/utils/` | 工具函数库 |

### 2.2 backend/ - 后端项目
| 目录 | 作用 |
|------|------|
| `app/` | Flask 应用核心，包含工厂函数和扩展注册 |
| `models/` | SQLAlchemy 数据模型，一个文件对应一个表 |
| `api/` | Flask Blueprint 路由定义，一个文件对应一个资源 |
| `services/` | 业务逻辑层，处理复杂业务，被 API 层调用 |
| `schemas/` | Marshmallow 序列化/反序列化，请求校验和响应格式化 |
| `middleware/` | 中间件，处理认证、错误等全局逻辑 |
| `utils/` | 通用工具函数 |
| `seeds/` | 种子数据，用于初始化数据库 |

### 2.3 分层架构说明
```
前端请求 → API 层（路由分发、参数校验）
         → Service 层（业务逻辑）
         → Model 层（数据访问）
         → Database

响应返回 ← API 层（格式化响应）
         ← Schema 层（序列化）
         ← Service 层（业务处理）
         ← Model 层（查询结果）
```

## 三、开发规范

### 3.1 文件命名规范
| 类型 | 规范 | 示例 |
|------|------|------|
| 前端组件 | PascalCase | `FoodCard.vue` |
| 前端工具 | camelCase | `formatDate.js` |
| 前端页面 | PascalCase | `FoodDetail.vue` |
| 后端模块 | snake_case | `food_service.py` |
| 数据库表 | snake_case | `food_items` |

### 3.2 代码组织原则
- **单一职责**：每个文件只做一件事
- **模块化**：相关功能放在同一目录
- **层次清晰**：API → Service → Model 三层分离
- **依赖方向**：上层依赖下层，禁止反向依赖
