# 菏泽美食推荐与城市漫游规划系统 - 标签体系设计

## 一、设计原则

### 1.1 标签体系目标
- 支撑推荐算法的特征提取
- 实现精准的用户画像
- 支持多维度的搜索和筛选
- 便于机器学习模型训练

### 1.2 设计原则
- **层次清晰**：分类 → 子类 → 具体标签
- **互斥性**：同一维度下标签互斥（如辣度只能选一个等级）
- **可扩展**：支持未来新增标签
- **标准化**：统一格式，便于程序处理

## 五、标签向量化设计

### 5.1 美食特征向量

```python
# 美食特征向量结构（20维）
food_vector = [
    # 口味特征（5维）
    spicy_level,      # 0-5
    sweet_level,      # 0-5
    salty_level,      # 0-5
    sour_level,       # 0-5
    umami_level,      # 0-5

    # 价格特征（1维）
    price_level,      # 1-5

    # 口感特征（7维，one-hot）
    is_crispy,        # 酥脆
    is_soft,          # 软糯
    is_smooth,        # 爽滑
    is_tender,        # 鲜嫩
    is_chewy,         # Q弹
    is_fine,          # 细腻
    is_firm,          # 筋道

    # 场景特征（4维，multi-hot）
```

### 5.3 相似度计算

```python
import numpy as np
from sklearn.metrics.pairwise import cosine_similarity

def calculate_similarity(user_vector, food_vector):
    """
    计算用户与美食的相似度
    使用余弦相似度，值越接近1表示越匹配
    """
    user_array = np.array(user_vector).reshape(1, -1)
    food_array = np.array(food_vector).reshape(1, -1)
    return cosine_similarity(user_array, food_array)[0][0]
```

## 七、标签维护规范

### 7.1 新增标签流程
1. 评估标签的必要性和使用频率
2. 确定标签所属维度
3. 更新标签配置文件
4. 更新数据库结构（如需要）
5. 更新推荐算法特征向量

### 7.2 标签质量控制
- 定期清理无用标签
- 合并语义重复标签
- 统一标签命名规范
- 监控标签使用统计
