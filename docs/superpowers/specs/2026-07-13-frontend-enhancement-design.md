# 前端增强与真实路径规划 — 设计规格

## 概述
修复地图图层开关bug，新增搜索功能，为美食增加全面的多维度标签体系，集成营业时间与实时营业状态判断，使用高德驾车路径API替代直线距离。

## 功能清单

### 1. 修复图层开关bug
- `showFoodMarkers()`/`showSpotMarkers()` 在重建标记时检查toggle复选框状态
- toggle状态缓存为全局变量，避免重建后标记忽略开关状态
- 文件: `map.js`

### 2. 搜索功能
- 侧边栏顶部新增搜索面板（输入框+类型切换+排序选择）
- 前端实时过滤 foodData + spotData（名称、地址、标签模糊匹配）
- 拼音首字母搜索支持
- 搜索结果列表，点击定位到地图标记
- 匹配标记高亮，未匹配标记降低透明度
- 文件: `index.html`, `map.js`, `style.css`

### 4. 营业时间与实时状态
- 新增 `opentime` 字段到 food.txt 格式中
- 高德API获取 `biz_ext.opentime`，对已有数据按分类推断默认营业时间
- 前端JS实时计算：解析opentime → 对比当前时间 → 判断营业状态
- 标记展示：营业中正常 / 即将打烊黄色边框 / 已打烊灰色半透明
- 侧边栏"仅显示营业中"切换按钮
- 推荐路线过滤已打烊美食

### 5. 高德驾车路径API
- `recalc_roads.py` 中集成高德驾车路径规划API替代Haversine
- 对同一区县内的美食间连接调用 `restapi.amap.com/v3/direction/driving`
- 跨区连接继续使用Haversine×1.3系数（减少API调用量）
- 超过200km的连接不调用API（超出驾车场景）
- 每个请求间隔0.5秒，失败降级到Haversine

### 6. 前端非必要不使用emoji
- 美食标记用CSS生成的颜色圆形+首字替代emoji
- 景点标记用CSS图标替代🏛
- 搜索面板、标签展示等使用纯CSS图标

## 数据格式变更

### food.txt 新格式
```
id|name|lng|lat|price|score|category|address|opentime|tags
```
- `opentime`: 营业时间字符串，如 `06:00-14:00,17:00-02:00`，未知时为 `-`
- `tags`: 逗号分隔，含12维度标签

## 错误处理
- 驾车API请求失败：降级到Haversine距离
- 营业时间解析失败：显示"营业时间未知"，默认视为营业中
- 搜索无结果：显示"未找到匹配项"
