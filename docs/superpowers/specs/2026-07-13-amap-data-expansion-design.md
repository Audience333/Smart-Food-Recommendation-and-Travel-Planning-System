# 高德API数据扩展 — 设计规格

## 概述
使用高德地图 API 获取菏泽市真实美食门店和景点数据，扩展现有的63条美食和20个景点数据集。

## 数据管道流程
```
高德API (按9个区县 + 多关键词搜索)
    ↓
expand_data.py (增强版)
    ├── 读取现有 data/*.txt → 已有ID和名称集合
    ├── 遍历9个区县，多关键词搜索美食和景点
    ├── 去重（名称 + 坐标匹配）
    ├── 自动分配新ID，生成TXT管道分隔格式
    └── 追加写入 data/food.txt 和 data/spot.txt
    ↓
recalc_roads.py → 重新计算所有节点间道路
    ↓
gen_web_json.py → TXT → JSON 转换
    ↓
web/ 前端自动显示新增数据
```

## 搜索参数

| 参数 | 值 |
|------|-----|
| 区县 | 牡丹区、单县、曹县、郓城、巨野、东明、定陶、成武、鄄城 |
| 美食关键词 | 火锅,烧烤,面馆,川菜,鲁菜,小吃,快餐,甜品,饮品,海鲜,自助餐,农家菜,特色菜 |
| 美食类型码 | 050000\|051000\|052000 |
| 景点关键词 | 公园,景区,博物馆,文化,寺庙,展览 |
| 景点类型码 | 110000\|110100\|110200 |
| 每页条数 | 20条，每个搜索翻2页 |
| 请求间隔 | 0.3秒，失败重试2次 |

## 去重规则
- 精确名称匹配跳过
- 坐标距离 < 200米 且 名称相似度 > 0.7 视为重复
- 同连锁店不同分店（坐标>200米）保留

## ID分配
- 美食：从 164 开始
- 景点：从 21 开始

## 数据字段
- 美食新增 `address` 字段：`id|name|lng|lat|price|score|category|address|tags`
- 景点保持现有格式：`id|name|description|address|lng|lat|type|ticketInfo|openingTime|recommendDuration|bestSeason|score|tags`

## 错误处理
- 坐标为空跳过；评分异常置默认值
- 每个请求间隔0.3秒防限流
- 失败重试2次

## 兼容性
- 新增字段追加在现有固定字段之后
- 解析器需支持可选新字段，空字段用 `-` 占位
