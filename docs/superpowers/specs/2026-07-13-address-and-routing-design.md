# 地址补全与实时路线规划 — 设计规格

## 概述
补全所有缺失的POI地址数据，使用高德驾车/步行API实现真实路径规划，提供用户自由选择起点/途经点/终点的交互面板。

## 功能清单

### 1. 后端地址补全
- 新建 `tools/fill_addresses.py` 脚本
- 读取 food.txt/spot.txt，对 address 字段为 "-" 或空的条目调用高德逆地理编码API
- 坐标去重：相同坐标只请求一次，结果共享
- 请求间隔0.1秒，失败重试1次，仍失败标记 "无法获取"
- 写回 TXT 文件，运行 gen_web_json.py 更新 JSON
- 预计补全约120个缺失地址（63旧美食 + 53新增美食 + 3新增景点 - 去重）

### 2. 路线规划面板
- 侧边栏新增路线规划面板
- 通行方式切换：驾车 / 步行
- POI下拉选择器：起点、终点，支持文字过滤搜索
- 途经点列表：支持添加（最多5个）、删除、上下移动排序
- [规划路线] 按钮触发实时API调用
- [清空路线] 清除当前路线

### 3. 实时路径API集成
- 前端JS调用高德驾车API: `/v3/direction/driving`
- 前端JS调用高德步行API: `/v3/direction/walking`
- 逐段计算：起点→途经→...→终点，每段独立API请求
- 返回数据：距离(m)、时间(s)、过路费(元)、路径坐标(polyline)
- 汇总显示：总距离、总时间、总过路费、分段详情

### 4. 路径渲染
- 多段折线用渐变色区分（红→橙→紫）
- 每段标签显示"X km / Y min / ¥Z"
- 途经点标记（圆形编号）
- 可拖拽途经点调整顺序（需加载AMap.DragRoute插件）

### 5. 废弃预生成路线
- 不再依赖 route.json
- route.json 保留为示例但不再使用

## 错误处理
- 单段API失败：降级到Haversine直线 + 标记"无法获取实际路径"
- 全部失败：弹窗提示
- 距离>500km：提示分日出行
- 地址API失败：重试后标记"无法获取"

## 数据格式不变
- food.txt/spot.txt 格式不变，仅 address 字段从 "-" 变为实际地址
