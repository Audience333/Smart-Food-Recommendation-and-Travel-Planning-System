## 二、前端技术选型

### 2.1 核心框架
| 技术 | 版本 | 选择理由 |
|------|------|----------|
| **Vue 3** | 3.4+ | 渐进式框架，学习曲线平缓， Composition API 更灵活 |
| **Vite** | 5.0+ | 极速热更新，开发体验优秀 |
| **Vue Router** | 4.0+ | Vue 官方路由，与 Vue 3 完美集成 |
| **Pinia** | 2.0+ | Vue 3 官方状态管理，替代 Vuex |

### 2.2 UI组件库
| 技术 | 版本 | 选择理由 |
|------|------|----------|
| **Element Plus** | 2.5+ | Vue 3 生态最成熟的 UI 库，组件丰富 |
| **Tailwind CSS** | 3.4+ | 原子化 CSS，快速开发自定义样式 |

### 2.3 地图方案
| 方案 | 选择理由 |
|------|----------|
| **高德地图 JS API** | 国内地图首选，POI 数据丰富，免费额度足够 |
| **Leaflet** (备选) | 开源方案，无 API 限制，适合离线场景 |

**推荐方案：高德地图 JS API 2.0**
- 免费额度：每日 5000 次调用（开发阶段足够）
- 优势：国内数据准确、POI 丰富、文档完善
- 集成方式：通过 `@amap/amap-jsapi-loader` 加载

### 2.4 其他依赖
| 库 | 用途 |
|----|------|
| **Axios** | HTTP 请求 |
| **ECharts** | 数据可视化（评分图表、统计） |
| **Day.js** | 日期处理 |

## 三、后端技术选型

### 3.1 核心框架
| 技术 | 版本 | 选择理由 |
|------|------|----------|
| **Python** | 3.10+ | 语法简洁，AI/ML 生态最强 |
| **Flask** | 3.0+ | 轻量级框架，快速开发，易于扩展 |
| **Flask-RESTful** | 0.3.10 | RESTful API 快速开发 |
| **Flask-CORS** | 4.0+ | 处理跨域请求 |

### 3.2 数据库 ORM
| 技术 | 版本 | 选择理由 |
|------|------|----------|
| **SQLAlchemy** | 2.0+ | Python 最强大的 ORM，支持多种数据库 |
| **Flask-Migrate** | 4.0+ | 数据库迁移工具 |

### 3.3 其他依赖
| 库 | 用途 |
|----|------|
| **PyJWT** | JWT Token 认证 |
| **Flask-Limiter** | API 限流 |
| **Marshmallow** | 数据序列化和验证 |

### 3.4 选择 Flask 而非 Django 的理由
1. **轻量灵活**：项目不需要 Django 的"全家桶"
2. **快速迭代**：Flask 更适合中小型项目
3. **AI 集成**：Flask 与 Python ML 库集成更自然
4. **学习成本**：团队更容易上手

## 四、数据库选型

### 4.1 主数据库：MySQL 8.0
| 考量 | 说明 |
|------|------|
| **选择理由** | 成熟稳定、社区活跃、免费开源 |
| **版本** | 8.0+（支持 JSON 类型、窗口函数） |
| **字符集** | utf8mb4（支持 emoji 和中文） |

### 4.2 缓存数据库：Redis
| 考量 | 说明 |
|------|------|
| **选择理由** | 高性能缓存，减少数据库压力 |
| **用途** | 热点数据缓存、会话存储、排行榜 |

### 4.3 数据库部署方案
- **开发阶段**：本地 MySQL + SQLite（快速启动）
- **生产阶段**：云数据库（阿里云 RDS / 腾讯云）

## 五、推荐算法模块设计

### 5.1 算法框架
```
推荐系统架构：
┌─────────────────────────────────────────┐
│              推荐引擎                    │
├──────────┬──────────┬───────────────────┤
│ 内容推荐 │ 协同过滤 │   混合推荐        │

```

### 5.2 技术栈
| 技术 | 用途 |
|------|------|
| **NumPy** | 向量计算 |

## 七、API 设计规范

### 7.1 RESTful API 规范
```
GET    /api/foods          # 获取美食列表
GET    /api/foods/:id      # 获取美食详情
POST   /api/foods          # 创建美食（管理员）
PUT    /api/foods/:id      # 更新美食（管理员）
DELETE /api/foods/:id      # 删除美食（管理员）

GET    /api/spots          # 获取景点列表
GET    /api/spots/:id      # 获取景点详情

GET    /api/users/me       # 获取当前用户信息
PUT    /api/users/me       # 更新用户信息

POST   /api/comments       # 发表评论
GET    /api/comments/:type/:id  # 获取评论列表

POST   /api/favorites      # 添加收藏
DELETE /api/favorites/:id  # 取消收藏

GET    /api/recommend      # 获取推荐结果
POST   /api/route/plan     # 生成路线规划
```

### 7.2 响应格式
```json
{
  "code": 200,
  "message": "success",
  "data": { ... }
}
```
