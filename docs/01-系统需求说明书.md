5. 体现数据结构在实际问题中的应用价值

### F1：数据管理模块

**功能描述**：从 CSV/TXT 文件加载美食和景点数据到内存，支持增删改查。

**业务场景**：
- 系统启动时从 `data/food.csv` 加载美食数据
- 系统启动时从 `data/spot.csv` 加载景点数据
- 用户可新增、修改、删除记录
- 退出时将修改写回文件

**数据结构设计**：
```
┌──────────────────────────────────────────────────┐
│  DataManager                                      │
│  ┌────────────────────────────────────────────┐  │
│  │  vector<Food> foods         // 美食主存储   │  │
│  │  vector<Spot> spots         // 景点主存储   │  │
│  │  unordered_map<int, int>    // ID → 索引    │  │
│  └────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────┘
```

**Food 数据结构**：
```cpp
struct Food {
    int id;                      // 唯一标识
    string name;                 // 名称
    string category;             // 分类（小吃/正餐/汤类）
    string address;              // 地址
    double longitude, latitude;  // 经纬度
    string cuisine;              // 菜系
    string signature_dish;       // 招牌菜
    double price_min, price_max; // 价格区间
    double score;                // 评分
    int spicy_level;             // 辣度 0-5
    int sweet_level;             // 甜度 0-5
    int salty_level;             // 咸度 0-5
    int sour_level;              // 酸度 0-5
    string texture;              // 口感
    vector<string> tags;         // 标签列表
    string suitable_scene;       // 适合场景
    string target_group;         // 目标人群
};
```

**Spot 数据结构**：
```cpp
struct Spot {
    int id;
    string name;
    string description;
    string address;
    double longitude, latitude;
    string type;                 // 景点类型
    string ticket_info;          // 门票信息
    string opening_time;         // 开放时间
    double score;
    vector<string> tags;
};
```

**文件格式**（CSV）：
```
id,name,category,address,lng,lat,cuisine,signature_dish,price_min,price_max,score,spicy,sweet,salty,sour,texture,tags
1,菏泽烧牛肉,小吃,菏泽市牡丹区中华路,115.4806,35.2337,鲁菜,烧牛肉,30,50,4.5,0,0,3,0,鲜嫩,传统|牛肉|鲁菜|老字号
```

**输入**：CSV 文件路径
**输出**：内存中的数据集合
**复杂度要求**：加载 O(n)，按 ID 查找 O(1)

### F2：多键索引模块

**功能描述**：为美食和景点建立多个索引，支持按不同字段快速查询。

**业务场景**：
- 按名称精确查找（哈希表）
- 按分类筛选所有美食
- 按评分范围查询（BST 中序遍历）
- 按价格区间筛选

**数据结构设计**：
```
┌─────────────────────────────────────────────────────────┐
│  IndexManager                                            │
│                                                         │
│  主存储:  vector<Food>                                   │
│                                                         │
│  索引层:                                                 │
│  ┌────────────────────────────────────────────────────┐ │
│  │  unordered_map<int, size_t>      id_index          │ │
│  │  // ID → foods[] 下标，O(1) 精确查找                │ │
│  │                                                    │ │
│  │  unordered_map<string, vector<size_t>>             │ │
│  │      category_index                                │ │
│  │  // 分类 → 下标列表，O(1) 分类筛选                  │ │
│  │                                                    │ │
│  │  BST<double, vector<size_t>>                       │ │
│  │      score_index                                   │ │
│  │  // 评分 → 下标列表，O(log n) 范围查询              │ │
│  │  // 支持查询 "评分 >= 4.0 的所有美食"               │ │
│  │                                                    │ │
│  │  BST<double, vector<size_t>>                       │ │
│  │      price_index                                   │ │
│  │  // 价格 → 下标列表，O(log n) 范围查询              │ │
│  └────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────┘
```

**BST 实现要求**：
- 自平衡二叉搜索树（AVL 或红黑树）
- 支持范围查询：`range_query(low, high)` 返回 [low, high] 范围内所有节点
- 支持插入、删除时自动维护索引

**输入**：查询条件（字段名 + 值/范围）
**输出**：符合条件的记录列表
**复杂度要求**：
- 按 ID 查找：O(1)
- 按分类查找：O(1)
- 按评分/价格范围：O(log n + k)，k 为结果数

### F3：前缀搜索模块（Trie 树）

**功能描述**：支持输入名称前缀，实时返回匹配的美食/景点名称。

**业务场景**：
- 用户输入"单县"，自动提示"单县羊肉汤"、"单县烧饼"等
- 用户输入"曹"，提示"曹县烧牛肉"、"曹州牡丹园"等
- 支持拼音首字母搜索（可选扩展）

**数据结构设计**：
```
                    root
                   / | \
                  菏 曹 单
                 /   |   \
                泽   县   县
               /    / \    \
              烧  烧牛 烧   羊
             /   /     \    \
            牛  肉      饼   肉
           /
          肉
          ↓
        [Food索引: 1, 3]

每个节点存储:
  - children: unordered_map<char, TrieNode*>
  - is_end: bool
  - indices: vector<size_t>  // 匹配到的数据索引
```

**接口设计**：
```cpp
class Trie {
public:
    void insert(const string& key, size_t index);
    vector<size_t> search_prefix(const string& prefix);
    void remove(const string& key, size_t index);
};
```

**输入**：用户输入的前缀字符串
**输出**：匹配的数据索引列表
**复杂度要求**：插入 O(m)，前缀搜索 O(m + k)，m 为前缀长度，k 为结果数

### F4：空间查询模块（KD-Tree）

**功能描述**：基于经纬度进行空间查询，支持最近邻搜索和范围搜索。

**业务场景**：
- "距离我最近的美食" — K 近邻搜索
- "方圆 2 公里内的景点" — 范围搜索
- "从当前位置出发，附近有什么好吃的"

**数据结构设计**：
```
KD-Tree (k=2, 经纬度):
                    (115.48, 35.23)     ← 根节点，按 lng 分割
                    /              \
          (115.35, 35.10)    (115.55, 35.30)
          按 lat 分割         按 lat 分割
           /      \            /       \
         ...      ...        ...       ...

每个节点:
  - point: {lng, lat}
  - data_index: size_t    // 指向 foods[] 或 spots[] 的索引
  - left, right: KDNode*
  - split_axis: int       // 0=经度, 1=纬度
```

**接口设计**：
```cpp
class KDTree {
public:
    void build(const vector<Point2D>& points);
    // K 近邻搜索
    vector<pair<double, size_t>> knn(double lng, double lat, int k);
    // 范围搜索（圆形范围，单位：公里）
    vector<size_t> range_search(double lng, double lat, double radius_km);
private:
    double haversine(double lng1, double lat1, double lng2, double lat2);
};
```

**距离计算**：使用 Haversine 公式计算球面距离
```
a = sin²(Δlat/2) + cos(lat1) · cos(lat2) · sin²(Δlng/2)
c = 2 · atan2(√a, √(1-a))
d = R · c    (R = 6371 km)
```

**输入**：当前位置经纬度 + k 或半径
**输出**：最近的 k 个点 / 范围内的所有点
**复杂度要求**：KNN O(log n)，范围搜索 O(√n + k)

### F5：个性化推荐模块

**功能描述**：基于用户偏好向量与美食特征向量的余弦相似度，推荐最匹配的美食。

**业务场景**：
- 用户设置口味偏好（喜欢辣、中等消费、喜欢小吃）
- 系统计算偏好向量与所有美食向量的相似度
- 返回 Top-K 最匹配的美食

**数据结构设计**：
```
用户偏好向量 (16维):
┌─────────────────────────────────────────────────┐
│ spicy │ sweet │ salty │ sour │ umami │ price    │ ← 口味+价格 (6维)
│ crispy│ soft  │ smooth│tender│ chewy │ fine │firm│ ← 口感 (7维)
│ breakfast│ lunch│ dinner│ latenight            │ ← 场景 (4维)
└─────────────────────────────────────────────────┘

美食特征向量 (16维): 结构相同，数值来自数据文件

相似度计算:
  cosine_sim(A, B) = (A · B) / (|A| × |B|)
  A · B = Σ(a[i] * b[i])
  |A| = √(Σ a[i]²)
```

**推荐流程**：
```
用户偏好输入
    ↓
构造偏好向量 (vector<double>, 16维)
    ↓
遍历所有美食，计算余弦相似度
    ↓
使用最小堆 (priority_queue) 维护 Top-K
    ↓
输出推荐结果（按相似度降序）
```

**接口设计**：
```cpp
class Recommender {
public:
    // 设置用户偏好
    void set_preference(const UserPreference& pref);
    // 获取 Top-K 推荐
    vector<RecommendResult> recommend(int k = 10);
    // 基于指定美食找相似
    vector<RecommendResult> similar(int food_id, int k = 5);
private:
    double cosine_similarity(const vector<double>& a, const vector<double>& b);
    vector<double> build_food_vector(const Food& food);
    vector<double> build_user_vector(const UserPreference& pref);
};
```

**输入**：用户偏好 / 指定美食 ID + k
**输出**：推荐结果列表（美食 + 相似度分数）
**复杂度要求**：O(n log k)，n 为美食总数，k 为推荐数量

### F6：城市漫游路线规划模块

**功能描述**：基于图结构，规划从起点出发经过多个景点/美食点的最优路线。

**业务场景**：
- 用户选择起点和若干途经点（景点 + 美食）
- 系统规划最短路径路线
- 输出路线顺序和总距离

**数据结构设计**：
```
图表示（邻接表）:
┌─────────────────────────────────────────────────┐
│  CityGraph                                       │
│                                                  │
│  节点: 景点/美食点 (共 N 个)                       │
│  边: 两点间的距离 (Haversine 计算)                 │
│                                                  │
│  adj[0] → [{1, 2.3}, {3, 1.5}]  // 节点0的邻居   │
│  adj[1] → [{0, 2.3}, {2, 3.1}]                  │
│  adj[2] → [{1, 3.1}, {3, 0.8}]                  │
│  adj[3] → [{0, 1.5}, {2, 0.8}]                  │
│                                                  │
│  无向带权图，权 = 两点距离(km)                     │
└─────────────────────────────────────────────────┘
```

**算法选择**：

| 场景 | 算法 | 说明 |
|------|------|------|
| 两点间最短路 | Dijkstra | 单源最短路径 |
| 所有点间最短路 | Floyd | 全源最短路径，预处理后 O(1) 查询 |
| 经过指定点的最优路线 | 状态压缩 DP / 回溯 | 类似 TSP 问题 |
| 连通性判断 | BFS / DFS | 判断两点是否可达 |

**路线规划流程**：
```
用户选择起点 S 和途经点 {A, B, C}
    ↓
用 Floyd 预处理所有点对间最短距离
    ↓
构建途经点的完全图（距离矩阵）
    ↓
用状态压缩 DP / 回溯求最短哈密顿路径
    ↓
回溯路径，输出完整路线
    ↓
格式: S → A → C → B，总距离: 12.5km
```

**接口设计**：
```cpp
class CityGraph {
public:
    // 从数据构建图（自动计算相邻节点的边）
    void build(const vector<Point2D>& locations, double connect_radius);
    // Dijkstra: 单源最短路径
    vector<int> dijkstra(int src, int dst);
    // Floyd: 全源最短路径
    void floyd();
    double get_distance(int u, int v);
    // TSP 路线规划
    RouteResult plan_route(int start, const vector<int>& waypoints);
private:
    vector<vector<pair<int, double>>> adj;  // 邻接表
    vector<vector<double>> dist;            // Floyd 距离矩阵
};
```

**输入**：起点 + 途经点列表
**输出**：最优路线（点序列 + 总距离）
**复杂度要求**：
- Dijkstra: O((V + E) log V)
- Floyd: O(V³)
- TSP(状态压缩): O(2^n · n²)

### F7：排序系统

**功能描述**：支持对美食/景点按不同字段排序。

**业务场景**：
- 按评分从高到低排序

**算法实现**：

| 排序算法 | 适用场景 | 复杂度 |
|----------|----------|--------|

```cpp

template<typename T, typename Compare>
vector<T> top_k(const vector<T>& arr, int k, Compare comp);
```

### F8：标签系统与倒排索引

**功能描述**：基于标签的倒排索引，支持按标签快速检索。

**业务场景**：
- 查询所有带"传统"标签的美食
- 查询同时带"早餐"和"平价"标签的美食
- 标签的交集 / 并集查询

**数据结构设计**：
```
倒排索引:
┌────────────┬─────────────────────┐
│  标签       │  包含该标签的数据ID  │
├────────────┼─────────────────────┤
│ "传统"     │  {1, 3, 7, 12}      │
│ "早餐"     │  {2, 5, 7, 15}      │
│ "平价"     │  {2, 4, 5, 8}       │
│ "牛肉"     │  {1, 3, 9}          │
│ "鲁菜"     │  {1, 3, 6, 10}      │
└────────────┴─────────────────────┘

底层: unordered_map<string, set<int>>

交集查询 "传统" ∩ "早餐" = {7}
并集查询 "传统" ∪ "早餐" = {1,2,3,5,7,12,15}
```

**接口设计**：
```cpp
class TagIndex {
public:
    void add(int id, const vector<string>& tags);
    void remove(int id, const vector<string>& tags);
    // 标签交集查询
    set<int> search_all(const vector<string>& tags);
    // 标签并集查询
    set<int> search_any(const vector<string>& tags);
    // 获取热门标签
    vector<pair<string, int>> get_top_tags(int k);
private:
    unordered_map<string, set<int>> inverted_index;
};
```

**输入**：标签列表 + 查询模式（交集/并集）
**输出**：符合条件的数据 ID 集合
**复杂度要求**：单标签查询 O(1)，交集/并集 O(min_set_size · num_sets)

## 三、数据结构与算法总览

### 3.1 数据结构使用清单

| 数据结构 | 用途 | 实现要求 |
|----------|------|----------|
| **动态数组** `vector<T>` | 美食/景点主存储 | 使用 STL 或自行实现 |
| **链表** `list<T>` | 操作历史记录、撤销功能 | 双向链表 |
| **哈希表** `unordered_map` | ID 索引、分类索引、倒排索引 | 使用 STL 或自行实现（开链法） |
| **BST / AVL 树** | 评分索引、价格索引（范围查询） | 自行实现，支持范围查询 |
| **Trie 树** | 名称前缀搜索 | 自行实现 |
| **KD-Tree** | 空间最近邻/范围查询 | 自行实现 |
| **图（邻接表）** | 城市路线网络 | 自行实现 |
| **最小堆 / 优先队列** | Top-K 推荐、Dijkstra | 使用 STL 或自行实现 |
| **栈** | 操作撤销、DFS | 使用 STL |
| **队列** | BFS 遍历 | 使用 STL |
| **集合** `set` | 标签倒排索引、去重 | 使用 STL |

### 3.2 算法使用清单

| 算法 | 用途 | 复杂度 |
|------|------|--------|
| **快速排序** | 通用排序 | O(n log n) |
| **归并排序** | 稳定排序 | O(n log n) |
| **堆排序** | Top-K | O(n log k) |
| **二分查找** | 有序数据查找 | O(log n) |
| **Dijkstra** | 单源最短路径 | O((V+E) log V) |
| **Floyd** | 全源最短路径 | O(V³) |
| **BFS / DFS** | 图遍历、连通性 | O(V + E) |
| **余弦相似度** | 推荐计算 | O(d)，d 为维度 |
| **Haversine** | 球面距离计算 | O(1) |
| **回溯 / 状态压缩 DP** | TSP 路线规划 | O(2^n · n²) |
| **KMP（可选）** | 字符串模式匹配 | O(n + m) |
| **集合交并运算** | 标签查询 | O(min(m,n)) |

## 七、开发阶段规划

### 阶段 1：基础框架与数据管理
**目标**：搭建项目骨架，实现文件读写和基础 CRUD
**涉及数据结构**：`vector`、文件 I/O
**输出**：能从 CSV 加载数据、显示列表

### 阶段 2：索引与搜索系统
**目标**：实现多键索引和前缀搜索
**涉及数据结构**：哈希表、BST、Trie
**输出**：支持按 ID/分类/评分/名称前缀快速查找

### 阶段 3：空间查询模块
**目标**：实现 KD-Tree 空间查询
**涉及数据结构**：KD-Tree
**输出**：支持最近邻和范围查询

### 阶段 4：推荐系统
**目标**：实现基于向量相似度的推荐
**涉及数据结构**：向量、优先队列
**输出**：支持个性化推荐和相似推荐

### 阶段 5：路线规划
**目标**：实现图结构和最短路径算法
**涉及数据结构**：邻接表图、Dijkstra、Floyd
**输出**：支持两点最短路和多点路线规划

### 阶段 6：标签系统与排序
**目标**：实现倒排索引和多种排序算法
**涉及数据结构**：倒排索引、堆
**输出**：支持标签查询和多条件排序

### 阶段 7：数据采集与完善
**目标**：完善数据采集工具，系统测试
**输出**：完整的数据集，系统稳定性验证
